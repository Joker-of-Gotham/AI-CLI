执行初始认证流程：
\begin{DoxyEnumerate}
\item 检查是否指定了认证类型
\item 如果未指定认证类型，直接返回
\item 尝试刷新认证令牌
\item 处理认证错误
\item 返回错误信息或 null（成功时）
\end{DoxyEnumerate}

参数：
\begin{DoxyItemize}
\item config\+: 应用程序配置对象
\item auth\+Type\+: 选定的认证类型
\end{DoxyItemize}

返回值：
\begin{DoxyItemize}
\item string \texorpdfstring{$\vert$}{|} null\+: 错误信息字符串或 null（成功时）
\end{DoxyItemize}\hypertarget{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md367}{}\doxysection{\texorpdfstring{认证流程}{认证流程}}\label{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md367}

\begin{DoxyEnumerate}
\item {\bfseries{认证类型检查}}
\begin{DoxyItemize}
\item 如果未指定 auth\+Type，函数直接返回 null
\item 这允许在未配置认证时正常启动
\end{DoxyItemize}
\item {\bfseries{认证执行}}
\begin{DoxyItemize}
\item 调用 config.\+refresh\+Auth(auth\+Type) 方法
\item 这会触发相应的认证流程
\item 不在控制台显示登录成功消息（有意省略）
\end{DoxyItemize}
\item {\bfseries{错误处理}}
\begin{DoxyItemize}
\item 捕获认证过程中抛出的异常
\item 将错误信息格式化为用户友好的字符串
\item 返回错误信息而不是抛出异常
\end{DoxyItemize}
\end{DoxyEnumerate}\hypertarget{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md368}{}\doxysection{\texorpdfstring{错误处理}{错误处理}}\label{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md368}

\begin{DoxyItemize}
\item 使用 get\+Error\+Message 工具函数获取一致的错误信息格式
\item 不中断应用程序启动流程，而是返回错误信息供调用方处理
\item 适用于启动时的非阻塞认证
\end{DoxyItemize}\hypertarget{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md369}{}\doxysection{\texorpdfstring{依赖关系}{依赖关系}}\label{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md369}

\begin{DoxyItemize}
\item 依赖 @google/gemini-\/cli-\/core 的 Config 类型和 get\+Error\+Message 函数
\item 依赖 @google/gemini-\/cli-\/core 的 Auth\+Type 类型
\end{DoxyItemize}\hypertarget{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md370}{}\doxysection{\texorpdfstring{使用场景}{使用场景}}\label{md_packages_2cli_2src_2core_2auth_8ts_autotoc_md370}
